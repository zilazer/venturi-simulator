# 项目路线图

本文档把当前对话中的需求沉淀为 GitHub 仓库层面的项目整理和后续开发方向。

## 已完成

- 单文件静态网页 `index.html`，可通过 GitHub Pages 发布。
- 仿真页：自动推荐文丘里几何和出口混合状态。
- 目标反算页：输入目标进水量和进气量，搜索可吸气方案。
- 手动校核页：输入完整几何参数，判断吸气、背压、逆流和空化状态。
- 深度计算页：1D 多节点沿程压力、速度、含气率和空化裕度计算。
- 工程口诀页：整理文丘里和微纳米气泡模块实验经验。
- 入口曲线页：比较锥形入口与曲线漏斗入口，并导出曲线/STP。
- 压力单位切换：`kPa`、`MPa`、`kgf/cm^2`。
- 进气量单位切换：`L/min`、`mL/min`。
- 中文/EN 语言切换。
- 各计算页固定状态栏。
- STP 与 Markdown 计算步骤导出。

## 近期建议

- 为典型低流量工况增加固定验算样例，例如 `入口直径 3.5 mm`、`进水 0.6 L/min`、`进气 < 1 L/min`。
- 增加更多公式旁注，明确每个经验系数的适用范围和默认值。
- 给第三方模型参数增加“恢复默认值”和“导入/导出参数配置”能力。
- 增加实验记录表单，用于保存实测流量、背压、吸气量和气泡状态。
- 增加发布前检查流程，确保每次推送后 GitHub Pages 页面可用。

## 后续增强

- 接入实验数据标定，按实测数据修正损失系数和进气孔流量系数。
- 增加微纳米气泡相关指标，例如停留时间、能量密度、估算剪切强度和白浊维持时间。
- 增加更多喷射器/曝气器模型，区分不同应用场景的默认系数。
- 增加方案保存、导入、导出和对比功能。
- 增加完整 PDF/HTML 报告导出。
- 增加 CAD 参数化模型生成，减少简化 STP 与制造模型之间的差异。

## 建议 GitHub Issues

1. 增加低流量标准验算样例库。
2. 增加第三方模型参数导入/导出。
3. 增加实验数据标定页面。
4. 增加参数方案保存和对比功能。
5. 增强 STP 导出几何精度和可制造性说明。
6. 增加自动化页面冒烟测试。
7. 增加 README 英文摘要。

## 发布流程建议

1. 本地修改 `index.html` 或文档。
2. 打开本地页面完成基础验收。
3. 更新 `CHANGELOG.md`。
4. 提交并推送到 `main`。
5. 打开 GitHub Pages URL 做线上验收。
6. 若发现问题，优先修复后再继续新增功能。
